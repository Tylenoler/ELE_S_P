


# 使用times
`times` 变量在这个代码中是一个**基于时间周期的多功能计数器**，它通过简单的取模运算实现了多个不同时间间隔的任务调度。以下是详细逻辑分析：

### 核心逻辑
1. **循环基准**：每次主循环末尾有 `delay_ms(10)`，即每 10ms 执行一次循环
2. **计数器递增**：`times++` 每次循环增加 1
3. **时间计算**：`times` 值 × 10ms = 实际经过时间

### 时间周期控制
通过 `times % N == 0` 实现周期性触发：

| 任务 | 判断条件 | 循环次数 | 实际时间间隔 | 功能 |
|------|----------|----------|--------------|------|
| LED 闪烁 | `times%30==0` | 30 次 | 30 × 10ms = 300ms | 翻转 LED0 状态（系统运行指示） |
| 输入提示 | `times%200==0` | 200 次 | 200 × 10ms = 2s | 打印"请输入数据..."提示 |
| 系统信息 | `times%5000==0` | 5000 次 | 5000 × 10ms = 50s | 打印开发板信息 |

### 工作流程示例
假设系统启动后 `times=0`：
```
时间点   times值  触发任务
--------------------------------
0ms     0        LED闪烁(0%30=0) 
               输入提示(0%200=0)
               系统信息(0%5000=0)
               
10ms    1        无
20ms    2        无
...     ...      ...
300ms   30       LED闪烁(30%30=0)
310ms   31       无
...     ...      ...
2000ms  200      输入提示(200%200=0)
2010ms  201      无
...     ...      ...
50000ms 5000     系统信息(5000%5000=0)
```

### 关键特性
1. **多任务调度**：单计数器管理多个不同周期的任务
2. **时间基准**：10ms 延时提供精确计时基础
3. **同步触发**：
   - 当 `times` 是多个周期的公倍数时（如 times=0），多个任务会同时触发
   - 示例：当 times=6000 时（6000%30=0, 6000%200=0），LED闪烁和输入提示同时触发

### 计数器溢出处理
- `times` 是 u16 类型（0-65535）
- 最大持续时间：65535 × 10ms ≈ 655.35 秒 ≈ 10.9 分钟
- 溢出后自动归零：65535 + 1 = 0
- 不影响周期性：取模运算在溢出后仍正常工作

### 实际应用意义
1. **LED 闪烁（300ms）**：肉眼可见的闪烁效果，指示系统正常运行
2. **输入提示（2s）**：适度提醒用户操作，避免频繁刷屏
3. **系统信息（50s）**：长时间运行时的状态确认，展示设备信息

这种设计是嵌入式系统中常见的**轻量级定时任务实现**，避免了使用硬件定时器中断的复杂性，适合对时间精度要求不高的场景。
